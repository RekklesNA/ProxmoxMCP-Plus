## Why this release matters

`v0.2.0` makes the project easier to trust, easier to run, and easier to explain.

This release is aimed at LLM, AI agent, homelab, and self-hosted users who want one Proxmox control plane for both `MCP` and `OpenAPI`.

## User-visible highlights

- real Proxmox live E2E coverage for VM, LXC, snapshot, backup, restore, ISO, SSH, OpenAPI, and Docker flows
- source-vs-installed-package test isolation fix so `pytest` runs against the repository `src` tree
- container SSH execution path verified on a live lab
- Docker OpenAPI `/health` and schema path verified
- clearer quick-start guidance for Proxmox API token, bridge, DNS, template, and SSH setup
